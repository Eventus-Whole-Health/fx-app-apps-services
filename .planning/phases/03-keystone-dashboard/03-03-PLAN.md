  <name>Task 1: Create schedule form drawer with create/edit/delete functionality</name>
  <files>
    keystone-platform/frontend/src/views/scheduler/components/ScheduleFormDrawer.tsx
  </files>
  <action>
    **ScheduleFormDrawer** — Create/edit form using the same right-side Sheet pattern as ExecutionHistoryDrawer:
    - Props:
      - `mode: 'create' | 'edit'`
      - `service: SchedulerService | null` (populated for edit, null for create)
      - `open: boolean`
      - `onOpenChange: (open: boolean) => void`
      - `onSuccess: () => void` (callback to refresh parent data after mutation)
    - Use Sheet/SheetContent/SheetHeader from `@/components/ui/sheet`
    - SheetContent side="right" with `sm:max-w-md` width
    - Header: "Create Schedule" or "Edit Schedule" based on mode, with subtitle showing function app name in edit mode

    **Form fields** (use shadcn Input, Select, Label, Switch):
    - **Function App** (text input) — required for create, readonly for edit (shown as disabled input)
    - **Service Name** (text input) — required for create, readonly for edit
    - **Trigger URL** (text input, type="url") — required, validated to start with "https://"
    - **Frequency** (Select dropdown with presets per user decision):
      - Options: "hourly", "daily", "weekly", "monthly"
      - No custom cron expression — locked decision
    - **Schedule Config** (conditional fields based on frequency):
      - daily: Time picker (hour:minute Select dropdowns, 24h format)
      - weekly: Day-of-week Select (Monday-Sunday) + Time picker
      - monthly: Day-of-month Select (1-28) + Time picker
      - hourly: No additional config needed
      - Store as JSON string in schedule_config field: `{"time": "08:00"}`, `{"day": "monday", "time": "14:00"}`, `{"day_of_month": 15, "time": "06:00"}`
    - **Max Retries** (number input, 0-10, default 3)
    - **Max Execution Minutes** (number input, 5-120, default 30)
    - **Active** (Switch toggle) — default true for create, current value for edit

    **Validation**:
    - Use local state for form errors (no external form library — keep it simple)
    - Required fields: function_app, service, trigger_url, frequency
    - trigger_url must start with "https://"
    - Show `FormMessage` component below invalid fields with red error text

    **Submit**:
    - Create: Call `createSchedule(formData)` from scheduler-api.ts
    - Edit: Call `updateSchedule(service.id, changedFields)` — only send fields that differ from original
    - Show loading spinner on submit button while request in flight
    - On success: show brief green success toast (or just close drawer), call `onSuccess()` to refresh
    - On error: show error message at top of form in red Alert component

    **Delete** (edit mode only):
    - "Delete Schedule" button at bottom of form in destructive style (red outline)
    - Clicking opens AlertDialog confirmation per user decision: "Are you sure you want to delete this schedule? This action will soft-delete the schedule — it can be restored by an admin."
    - Confirm calls `deleteSchedule(service.id)`
    - On success: close drawer, call `onSuccess()`

    **Pre-populate for edit mode**:
    - Parse `schedule_config` JSON to populate time/day fields
    - Set all form fields from `service` prop when mode is 'edit'
    - Use `useEffect` to reset form when `service` changes
  </action>

    - Form renders in edit mode with fields populated from service data
    - Frequency dropdown shows exactly 4 options: hourly, daily, weekly, monthly
    - Schedule config fields change dynamically based on selected frequency
    - Delete button only appears in edit mode and shows confirmation dialog
    - Validation prevents submission of empty required fields
    - TypeScript compilation: `cd keystone-platform/frontend && npx tsc --noEmit --pretty 2>&1 | head -30`
  </verify>
